Cacheer Monitor — API & Usage

Reference for the local dashboard endpoints and CLI usage.

Endpoints

GET /api/health

Returns { ok: true } if the server is running.

GET /api/config

Shows the current events file path and its origin (env, dotenv, or default).

GET /api/metrics?namespace=foo

Aggregated metrics with optional namespace filter. Includes hit rate, latency (avg, p95, p99), drivers, namespaces, types, and top keys.

GET /api/events?limit=200&namespace=foo

Returns recent events with optional limit and namespace parameters.

POST /api/events/clear

Rotates and truncates the current events file. Intended for local/dev use.

CLI

$ php bin/cacheer-monitor serve --host=127.0.0.1 --port=9966
# Optional: choose a custom events file path
$ export CACHEER_MONITOR_EVENTS=Tests/cacheer-monitor.json

The dashboard starts at http://127.0.0.1:9966. The events file path can be provided via OS env (CACHEER_MONITOR_EVENTS), .env, or falls back to a temp file.

Generating Events

Use the included test scenarios to populate the dashboard:

$ php Tests/scenarios.php

This will emit a variety of events: hits, misses, putMany, tag/flushTag, TTL expirations, renew, increment/decrement, append, clear and flush.

Configuration

  • CACHEER_MONITOR_EVENTS: absolute or relative path to the JSONL events file (supports OS env and .env).
  • Relative paths are resolved against the package root.
  • Default path if none set: system temp directory, file cacheer-monitor.jsonl.
